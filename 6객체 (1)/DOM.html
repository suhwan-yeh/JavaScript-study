DOM(문서 객체 모델) 파싱 & 노드 분리 가이드

브라우저가 HTML 코드를 읽고 트리 구조로 변환하는 과정을 명확하게 정리했습니다.

1. DOM(Document Object Model)이란?

DOM은 '브라우저가 HTML 문서를 이해하고 조작하기 위해 메모리 상에 만들어 둔 객체들의 구조도(Tree)'입니다.

우리가 작성한 .html 파일은 단순한 '텍스트'에 불과하여 자바스크립트가 직접 이해할 수 없습니다. 따라서 브라우저는 HTML 문서를 읽어들인 뒤, 자바스크립트가 접근하고 수정할 수 있도록 태그, 속성, 텍스트 하나하나를 '노드(Node)'라는 살아있는 객체로 변환하여 나뭇가지 형태(Tree)로 연결해 둡니다.

2. 파싱(Parsing) 4단계 과정
  • 변환 (Bytes to Characters):
    0과 1의 데이터를 지정된 인코딩(UTF-8)에 따라 우리가 아는 문자열로 변환합니다.
  • 토큰화 (Tokenization):
    문자열을 브라우저가 이해할 수 있는 의미 있는 최소 단위인 '토큰(Token)'으로 쪼갭니다.
  • 노드 생성 (Node Creation):
    쪼개진 토큰들을 기반으로 고유한 성질(요소, 속성, 텍스트)을 가진 '객체(Node)'를 메모리에 생성합니다.
  • 트리 조립 (Tree Construction):
    부모와 자식 관계를 계산하여 최종적인 계층형 트리(Tree)를 완성합니다.
3. 3가지 핵심 노드 분리 기준
종류 생성 기준 / 역할 HTML 예시
요소 노드
(Element)		 HTML의 '태그(Tag)'를 만났을 때 생성.
트리의 뼈대를 형성하는 부모 역할.		 <h1>, <p>
속성 노드
(Attribute)		 태그 안의 부가 설정값을 만났을 때 생성.
요소 노드에 종속된 부가 정보.		 id="main"
텍스트 노드
(Text)		 태그 사이에 있는 순수 글자를 만났을 때 생성.
자식을 가질 수 없는 트리의 끝단(잎사귀).		 "Hello"

🚀 DOM 트리 조립 시뮬레이터

<html lang="ko">
  <head>
    <title>DOM Test</title>
  </head>
  <body>
    <h1 id="main">Hello</h1>
  </body>
</html>
💡 '다음 파싱 단계' 버튼을 누르면 위 코드의 토큰이 노드로 변환되어 트리에 매달립니다.
